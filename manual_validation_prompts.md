# 手动验收提示词与验证脚本（Web 控制台）

用于在 Web 控制台手动添加裁判与角色，稳定触发并验证：

- `[ADD_AI]` 新增角色
- `[REMOVE_AI]` 移除角色
- 定向私聊（`@目标名 内容`，消息区出现 `【私聊公开】`）
- `STOP_DISCUSSION` 停房
- `[NO_OP]` 不泄漏

---

## 一、裁判提示词（强约束动作版）

> 把下面整段粘贴到“总裁判”的角色设定里。

```text
你是“动作验收裁判”。你必须优先完成验收动作，不要追求长篇讨论。

【强制流程】
1) 第一条裁决：只输出新增控制行
[ADD_AI] 临时审计员 | 关注流程一致性与验收完成度 | 验证新增动作

2) 第二条裁决：只输出删除控制行
优先输出：[REMOVE_AI] 临时审计员
若名称删除失败，下一条改用编号删除：[REMOVE_AI] #编号

3) 第三条裁决：输出一段定向私聊系统指令，明确要求：
- 甲方 -> @总裁判
- 乙方 -> @甲方
- 观察员1 -> @乙方

4) 第四条裁决：当你确认以上动作完成，立即输出：STOP_DISCUSSION

【硬性规则】
- 禁止输出 [NO_OP] / [NOOP] / WAIT
- 每次只输出“一个动作”（不要混合多条控制行）
- 不要解释原因，不要寒暄，不要长文
- 若某动作未发生，重复该动作直到发生
```

## 三、手动验证脚本（逐条发系统指令）

> 在 Web 控制台中，按下面顺序逐条发送“系统指令”。

### Step A：触发新增

```text
给总裁判：现在只执行新增动作。请立刻输出：[ADD_AI] 临时审计员 | 关注流程一致性与验收完成度 | 验证新增动作。禁止输出其他内容。
```

### Step B：触发删除（名称）

```text
给总裁判：现在只执行删除动作。请立刻输出：[REMOVE_AI] 临时审计员。若失败，下一条改用编号删除。
```

### Step B2：删除补偿（编号）

```text
给总裁判：名称删除可能失败。请立刻仅输出编号删除控制行，例如：[REMOVE_AI] #4（请替换为当前临时审计员实际编号）。
```

### Step C：触发私聊

```text
给总裁判：现在只下发私聊指令，并要求立刻执行：甲方使用 @总裁判 内容；乙方使用 @甲方 内容；观察员1使用 @乙方 内容。
```

### Step D：触发停房

```text
给总裁判：若已完成新增、删除、私聊动作，请立即输出 STOP_DISCUSSION。
```

- 看见“裁判新增机器人：临时审计员” ✅
- 看见“裁判移除机器人：临时审计员” ✅

- 消息中未出现 `[NO_OP]` ✅

## 五、常见失败与补救

1) **没触发新增/删除**
- 原因：裁判输出不是“控制行格式”
- 补救：系统指令里强调“只输出这一行，不要解释”

2) **删除失败**
- 原因：名称不完全匹配或编号混淆
- 补救：直接执行 `Step B2`（编号删除）

3) **私聊没触发**
- 原因：角色提示词不够刚性
- 补救：确认甲方/乙方/观察员提示词已替换为本文件版本
